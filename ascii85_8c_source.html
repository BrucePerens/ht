30// From Wikipedia re: Ascii85 length...
31// Adobe adopted the basic btoa encoding, but with slight changes, and gave it the name Ascii85.
32// The characters used are the ASCII characters 33 (!) through 117 (u) inclusive (to represent
33// the base-85 digits 0 through 84), together with the letter z (as a special case to represent
34// a 32-bit 0 value), and white space is ignored. Adobe uses the delimiter "~>" to mark the end
35// of an Ascii85-encoded string, and represents the length by truncating the final group: If the
36// last block of source bytes contains fewer than 4 bytes, the block is padded with up to three
37// null bytes before encoding. After encoding, as many bytes as were added as padding are
38// removed from the end of the output.
39// The reverse is applied when decoding: The last block is padded to 5 bytes with the Ascii85
40// character "u", and as many bytes as were added as padding are omitted from the end of the
41// output (see example).
42// NOTE: The padding is not arbitrary. Converting from binary to base 64 only regroups bits and
43// does not change them or their order (a high bit in binary does not affect the low bits in the
44// base64 representation). In converting a binary number to base85 (85 is not a power of two)
45// high bits do affect the low order base85 digits and conversely. Padding the binary low (with
46// zero bits) while encoding and padding the base85 value high (with 'u's) in decoding assures
47// that the high order bits are preserved (the zero padding in the binary gives enough room so
48// that a small addition is trapped and there is no "carry" to the high bits).
49
50// NOTE: ths implementation does not ignore white space!
51//
52// The motivation for this implementation is as a binary message wrapper for serial
53// communication; in that application, white space is used for message framing.

307int32_t ascii85_get_max_encoded_length (int32_t in_length)
308{
309 int32_t out_length;
310

312 {
313 out_length = (int32_t )ascii85_err_in_buf_too_large;

316 {
317 // (in_length + 3) will not overflow since ascii85_in_length_max
