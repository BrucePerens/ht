10// Log server. This emits a lot to a stream socket, which you can connect with via
11// telnet. Only one connection is supported. When you connect, logging is diverted
12// from the console to the socket, and it is restored to the console when you disconnect.
13// I wrote this to debug the Improv WiFi protocol, since logging to the same serial port
14// that would be running the Improv protocol was problematic.
